# 个人知识库 — 使用指南

## 你已拥有的能力

你的知识库位于 `E:\Rex\knowledge-base\`，通过 GitHub 私有仓库在办公室和家里电脑之间同步。

---

## 一、日常使用命令

打开 **Git Bash**（或任意终端），输入以下命令：

### 创建新笔记
```bash
kb new "笔记标题" --tags 标签1,标签2
```
例如：`kb new "Docker常用命令" --tags docker,运维`

### 导入 Word 文档
```bash
kb import "C:/Users/Administrator/Desktop/我的资料.docx"
```
导入后：
- 原始 Word 文件自动保存到 `_originals/` 文件夹
- 自动转换为 Markdown 格式放入 `notes/` 目录
- 原始文件随时可以下载打开

### 列出所有笔记
```bash
kb list                    # 列出全部
kb list --tag docker       # 按标签筛选
```

### 全文搜索
```bash
kb search "关键词"
```
遍历所有笔记内容，找到包含关键词的笔记并显示上下文。

### 智能语义搜索 + AI 回答（推荐！）
```bash
kb index                   # 首次使用先构建索引（只需做一次）
kb ask "关于Docker网络配置，我整理了哪些内容？"
```
系统会：
1. 语义理解你的问题
2. 找到最相关的笔记段落
3. 调用 Claude AI 整理成完整回答，并标注来源

### 同步到 GitHub（跨设备）
```bash
kb sync
```
自动执行 git add → commit → pull → push，完成后另一台电脑运行同样命令即可同步。

### 查看状态
```bash
kb status
```
显示笔记数量、索引状态、Git 同步状态。

## 三、笔记格式说明

每篇笔记使用 Markdown 格式，头部是元数据（称为 YAML frontmatter）：

```markdown
---
title: "笔记标题"
tags: [标签1, 标签2]
created: 2026-07-06
updated: 2026-07-06
links: ["../技术/另一篇笔记.md"]
source: "_originals/原始文件.docx"
---

# 笔记标题

正文内容...
```

`tags` 是标签，用于分类和筛选。
`links` 可以链接到其他笔记，建立知识之间的关联。

## 四、跨设备使用

### 办公室电脑（首次设置）
1. 在 GitHub 创建私有仓库（如 `my-kb`）
2. 将本地仓库关联到 GitHub：
   ```bash
   cd E:/Rex/knowledge-base
   git remote add origin https://github.com/你的用户名/my-kb.git
   git push -u origin main
   ```

### 家里电脑（首次设置）
```bash
git clone https://github.com/你的用户名/my-kb.git E:/Rex/knowledge-base
cd E:/Rex/knowledge-base
pip install python-docx scikit-learn
```
然后把 `E:/Rex/knowledge-base/_tools/kb.bat` 所在目录加入 PATH，或用完整路径调用。

### 日常同步
- 离开办公室前：`kb sync`（push 到 GitHub）
- 到家后：`kb sync`（pull 最新内容）

## 六、常见问题

**Q: 如何编辑已有笔记？**
A: 直接用任何编辑器打开 `notes/` 下的 .md 文件修改。推荐 VS Code（免费）。

**Q: 标签用中文还是英文？**
A: 都可以，随你喜欢。建议统一风格。

**Q: 索引需要经常重建吗？**
A: 新增笔记后运行 `kb index` 重建索引即可。目前笔记少，几秒完成。

**Q: 搜索不准确怎么办？**
A: 用 `kb search "关键词"` 做精确全文搜索，或用 Claude Code 对话方式查找。

## 七、进阶技巧

1. **笔记之间互相链接**：在 `links` 字段添加相对路径 `"../技术/某笔记.md"`
2. **自定义模板**：编辑 `_templates/note-template.md`
3. **批量导入**：把多个 .docx 文件放到某个文件夹，逐个 `kb import`
